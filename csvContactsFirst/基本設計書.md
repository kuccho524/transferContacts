# CSVデータ突合ツール 基本設計書

## 1. システム概要

### 1.1 システム名
CSVデータ突合ツール

### 1.2 目的
Google ContactsからエクスポートしたCSVデータから、連絡先登録用CSVとラベル作成用CSVの2種類のファイルを生成する。

### 1.3 対象ユーザー
企業の社内SE/情シス担当者

### 1.4 開発言語・環境
- **言語**: Python 3.x
- **標準ライブラリ**: csv, argparse, pathlib, tkinter, logging
- **実行環境**: Windows (MINGW64)

## 2. 機能要件

### 2.1 主要機能

#### 2.1.1 CSVデータ読み込み機能
- Google Contactsエクスポート形式のCSVファイルを読み込む
- 入力ファイルの文字コードは自動判定（UTF-8を想定）

#### 2.1.2 連絡先CSV生成機能
以下の項目を抽出して連絡先登録用CSVを生成する：
- 基本情報
  - FirstName（名）
  - LastName（姓）
  - HiraganaFirstName（名のひらがな）
  - HiraganaLastName（姓のひらがな）
  - OrganizationName（組織名）
  - OrganizationDepartment（部署名）

- メールアドレス（動的カラム）
  - PrimaryEmailLabel, PrimaryEmailAddress
  - SecondaryEmailLabel, SecondaryEmailAddress
  - ThirdEmailLabel, ThirdEmailAddress
  - （以降、データ件数に応じて増加）

- 電話番号（動的カラム）
  - PrimaryPhoneLabel, PrimaryPhoneNumber
  - SecondaryPhoneLabel, SecondaryPhoneNumber
  - ThirdPhoneLabel, ThirdPhoneNumber
  - （以降、データ件数に応じて増加）

#### 2.1.3 ラベルCSV生成機能
Labelsフィールドから以下の項目を生成する：
- PrimaryLabel（主ラベル）: 必ず "* myContacts" を設定
- SecondaryLabel（副ラベル1）: "* myContacts" 以外の最初のラベル
- ThirdLabel（副ラベル2）: "* myContacts" 以外の2番目のラベル
- FourthLabel, FifthLabel...（以降同様）: ラベル数に応じて動的に列を生成

ラベル分割ルール：
- " ::: " または ":::" で分割（スペースの有無を許容）
- "* myContacts" および "myContacts" を除外
- 残りのラベルをSecondaryLabel, ThirdLabel, FourthLabel...として順次割り当て
- 列名は序数形容詞を使用（Secondary, Third, Fourth, Fifth...）
- 50個以上のラベルにも対応可能
- PrimaryLabel列には全行に "* myContacts" を出力

#### 2.1.4 ファイル選択機能（フェーズ2で実装）
- ファイル選択ダイアログを使用して、入力ファイルと出力ファイルを選択可能
- コマンドライン引数との両立（引数がある場合は引数を優先）

#### 2.1.5 ログ出力機能（フェーズ2で実装）
- 処理内容をログファイルに記録
- ログレベル別の出力（DEBUG, INFO, WARNING, ERROR, CRITICAL）
- ログファイルの自動生成と保存
- エラー発生時の詳細情報をログに記録

ログ出力内容：
- **INFO**: 処理開始・終了、ファイルパス、処理件数
- **WARNING**: データ不整合、空欄データの検出
- **ERROR**: ファイル読み込みエラー、書き込みエラー
- **DEBUG**: 詳細な処理ステップ（開発・デバッグ用）

## 3. 非機能要件

### 3.1 出力ファイル形式
- **文字コード**: UTF-8 BOMなし
- **改行コード**: LF（Unix形式）
- **ファイル形式**: CSV（カンマ区切り）

### 3.2 データ処理
- 全レコードを処理対象とする
- 必須項目が欠けている場合でも空欄として出力
- エラーが発生した場合は標準エラー出力にメッセージを表示
- メールアドレスと電話番号のフィールドに " ::: " で区切られた複数の値がある場合：
  - 各値を個別のエントリとして抽出
  - 例: "070-0000-0001 ::: 090-0000-0002" → PrimaryPhoneNumber="070-0000-0001", SecondaryPhoneNumber="090-0000-0002"

### 3.3 ログ出力形式（フェーズ2以降）
- **ログファイル保存場所**: ./logs/convert_YYYYMMDD_HHMMSS.log
- **ログフォーマット**: `%(asctime)s - %(levelname)s - %(message)s`
- **文字コード**: UTF-8
- **ログレベル**: INFO（デフォルト）、コマンドライン引数で変更可能

### 3.4 パフォーマンス
- 1000件程度のレコードを10秒以内に処理

### 3.5 互換性
- Python 3.6以降で動作
- Windows環境で動作確認済み

## 4. システム構成

### 4.1 ファイル構成
```
csvContactsFirst/
├── convert_contacts.py      # メインスクリプト
├── logs/                    # ログファイル保存ディレクトリ（フェーズ2以降）
│   └── convert_YYYYMMDD_HHMMSS.log
├── 基本設計書.md            # 本ドキュメント
├── 詳細設計書.md            # 詳細設計書
├── テスト用エクスポートデータ.csv  # テストデータ（入力）
├── 連絡先テストデータ.csv    # 参考データ（連絡先）
└── ラベル作成テストデータ.csv # 参考データ（ラベル）
```

### 4.2 モジュール構成
- **メイン処理**: main()
- **ログ設定**: setup_logging()（フェーズ2で追加）
- **データ読み込み**: read_export_data()
- **連絡先CSV生成**: generate_contact_csv()
  - extract_emails_and_phones(): メール・電話抽出
  - get_ordinal_suffix(): 序数接尾辞生成
- **ラベルCSV生成**: generate_label_csv()
  - parse_labels(): ラベル分割処理
- **ファイル選択**: select_files()（フェーズ2で追加）

## 5. データフロー

```
[入力]
テスト用エクスポートデータ.csv
    ↓
[前処理] ログ設定（フェーズ2以降）
    ↓
[処理1] CSVデータ読み込み → ログ記録（INFO）
    ↓
[処理2] データ抽出・変換 → ログ記録（INFO/WARNING）
    ├→ [処理2-1] 連絡先データ生成 → ログ記録（INFO）
    └→ [処理2-2] ラベルデータ生成 → ログ記録（INFO）
    ↓
[出力] → ログ記録（INFO）
├→ 連絡先CSV
├→ ラベルCSV
└→ ログファイル（フェーズ2以降）
```

## 7. 使用方法

### 7.1 コマンドライン実行（フェーズ1）
```bash
python convert_contacts.py <入力ファイル> <連絡先CSV出力> <ラベルCSV出力>
```

**実行例**:
```bash
python convert_contacts.py "テスト用エクスポートデータ.csv" "連絡先.csv" "ラベル.csv"
```

### 7.2 ファイル選択ダイアログ実行（フェーズ2）
```bash
python convert_contacts.py
```
引数なしで実行すると、ファイル選択ダイアログが表示される。

### 7.3 ログレベル指定（フェーズ2）
```bash
python convert_contacts.py --log-level DEBUG <入力ファイル> <連絡先CSV出力> <ラベルCSV出力>
```
または
```bash
python convert_contacts.py --log-level INFO
```

**ログレベルオプション**:
- DEBUG: 詳細な処理ステップを記録
- INFO: 通常の処理情報を記録（デフォルト）
- WARNING: 警告のみ記録
- ERROR: エラーのみ記録

## 8. エラー処理

### 8.1 入力ファイルエラー
- ファイルが存在しない場合：エラーメッセージを表示（標準エラー出力＋ログ）して終了
- ファイルが読み込めない場合：エラーメッセージを表示（標準エラー出力＋ログ）して終了

### 8.2 データエラー
- 必須項目が欠けている場合：空欄として出力し、警告ログを記録して処理を継続
- 不正なCSV形式の場合：エラーメッセージを表示（標準エラー出力＋ログ）して終了

### 8.3 出力ファイルエラー
- 出力先ディレクトリが存在しない場合：エラーメッセージを表示（標準エラー出力＋ログ）して終了
- 書き込み権限がない場合：エラーメッセージを表示（標準エラー出力＋ログ）して終了

### 8.4 ログファイルエラー（フェーズ2以降）
- logsディレクトリが存在しない場合：自動作成
- ログファイル書き込みに失敗した場合：標準エラー出力のみで処理を継続
