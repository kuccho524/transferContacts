# 基本設計書
## 連絡先ラベル登録CSV生成ツール（Python版）


### 1. システム概要

#### 1.1 目的
Google Contactsの連絡先エクスポートデータと登録済み連絡先データを突合し、ラベル登録用のCSVファイルを生成するPythonアプリケーション

#### 1.2 システム構成
- **実行環境**: Windows（MINGW64）
- **実装言語**: Python 3.6以降
- **GUI**: tkinter（標準ライブラリ）
- **外部ライブラリ**: なし（Pure Python実装）

---

### 2. 機能要件

#### 2.1 入力ファイル
| ファイル名 | 説明 | 形式 | 必須列 |
|-----------|------|------|--------|
| 連絡先エクスポートデータ | Google Contactsエクスポートデータ | CSV（UTF-8） | First Name, Last Name, E-mail 1 - Value, Labels |
| 登録した連絡先のデータ | 登録済み連絡先の詳細情報 | CSV（UTF-8） | User, names.0.givenName, names.0.familyName, emailAddresses.0.value, resourceName |
| 登録したラベルのデータ | ラベル（グループ）のマスター情報 | CSV（UTF-8） | User, resourceName, name |

#### 2.2 出力ファイル
| 項目 | 仕様 |
|------|------|
| ファイル名 | contacts_labels_N.csv（Nはラベル数） |
| エンコーディング | UTF-8（BOMなし） |
| 改行コード | LF (\n) |
| ヘッダー行 | あり |
| 分割方式 | ラベル数ごとに別ファイル |

**ファイル分割仕様**:
- ラベル数ごとに個別のCSVファイルを生成
- ファイル名形式: `contacts_labels_1.csv`, `contacts_labels_2.csv`, `contacts_labels_3.csv`...
- 各ファイルには対応するラベル数の列のみ含める
- 例:
  - `contacts_labels_2.csv`: PrimaryLabel, SecondaryLabel の2列
  - `contacts_labels_3.csv`: PrimaryLabel, SecondaryLabel, ThirdLabel の3列

#### 2.3 出力CSV列定義
| 列名 | 説明 | 必須 | 備考 |
|------|------|------|------|
| TargetEmail | 対象GWSアカウントのメールアドレス | ✓ | 連絡先データのUser列から取得 |
| ContactID | 連絡先のID | ✓ | people/cXXXXXXXXXXXXXXXXXX形式 |
| PrimaryLabel | myContactsのラベルID | ✓ | contactGroups/myContacts |
| SecondaryLabel | ユーザー任意登録のラベルID（1つ目） | - | contactGroups/XXXXXXXX形式 |
| ThirdLabel | ユーザー任意登録のラベルID（2つ目） | - | 3つ以上のラベルがある場合 |
| FourthLabel | ユーザー任意登録のラベルID（3つ目） | - | 4つ以上のラベルがある場合 |
| FifthLabel, SixthLabel... | 以降も同様に序数形容詞で列を動的追加 | - | 実際のラベル数に応じて列が自動生成される |

**動的列生成仕様**:
- ラベル数は入力データから自動検出し、必要な列数を動的に生成
- 列名は序数形容詞を使用（Third, Fourth, Fifth, Sixth...）
- 50個以上のラベルにも対応可能
- 51個目以降は数値表記（51stLabel, 52ndLabel...）

### 3. 処理フロー

```
[GUI起動]
    ↓
[ファイル選択]
 - 連絡先エクスポートデータ
 - 登録した連絡先のデータ
 - 登録したラベルのデータ
 - 出力先
    ↓
[Phase 1: ファイル読み込み]
    ↓
[Phase 2: データ抽出]
 - エクスポートデータ: First Name, Last Name, E-mail 1 - Value, Labels
 - 連絡先データ: User, names.0.givenName, names.0.familyName,
                emailAddresses.0.value, resourceName (people/のみ)
 - ラベルデータ: User, resourceName, name
    ↓
[Phase 3: データ突合]
 - メールアドレス + 姓名で一致判定
 - 一致したデータを保持
 - 不一致データはスキップリストへ
    ↓
[Phase 4: ラベル解析]
 - エクスポートデータのLabels列をパース（:::区切り）
 - ラベルデータとマッピング
 - myContacts → PrimaryLabel
 - その他 → SecondaryLabel, ThirdLabel...
    ↓
[Phase 5: CSV出力]
 - UTF-8, LF改行で出力
 - スキップリストも同時出力
    ↓
[完了・ログ表示]
```

### 4. データ処理仕様

#### 4.1 データ一致判定ルール
**一致条件**: 以下の3項目がすべて一致すること
1. `First Name` = `names.0.givenName`
2. `Last Name` = `names.0.familyName`
3. `E-mail 1 - Value` = `emailAddresses.0.value`

**正規化ルール**:
- メールアドレス: 小文字に変換して比較
- 姓名: トリム後に完全一致で比較

#### 4.2 不一致データの処理
- **動作**: 処理をスキップ
- **出力**: スキップリストCSVファイルに出力
  - 列: 姓, 名, メールアドレス, 理由
- **ログ**: 警告ログに表示

#### 4.3 ラベル分類ルール
**Labels列のパース**:
- 区切り文字: `:::`
- 例: `Team A ::: * myContacts` → ['Team A', '* myContacts']

**ラベル分類**:
1. **PrimaryLabel**: `* myContacts`
   - ラベルデータから`name='myContacts'`のresourceNameを取得
   - 出力: `contactGroups/myContacts`

2. **SecondaryLabel以降**: ユーザー定義ラベル
   - `* `で始まらないラベル
   - ラベルデータから一致するnameのresourceNameを取得
   - 順序: エクスポートデータの出現順
   - 列名: SecondaryLabel, ThirdLabel, FourthLabel...
   - **動的対応**: 入力データに含まれるラベル数を自動検出し、必要な列を動的に生成
   - **制限なし**: 3個、4個、それ以上のラベルにも対応可能

#### 4.4 resourceNameフィルタリング
**条件**: `resourceName`が`people/`で始まるもののみ処理
- `people/c1234567890` → ✓ 処理対象
- `otherContacts/c1234567890` → ✗ 除外

### 7. エラーハンドリング

#### 7.1 ファイル読み込みエラー
- **発生条件**: ファイルが存在しない、読み込み権限がない
- **処理**: エラーダイアログ表示、処理中断

#### 7.2 CSV形式エラー
- **発生条件**: 必須列が存在しない、データ形式が不正
- **処理**: エラーダイアログ表示、不足列を明示

#### 7.3 文字コードエラー
- **発生条件**: UTF-8以外のエンコーディング
- **処理**: エラーダイアログ表示

#### 7.4 データ不一致
- **発生条件**: 姓名・メールアドレスが一致しない
- **処理**: スキップリストに追加、警告ログ表示

### 9. 非機能要件

#### 9.1 パフォーマンス
- **処理速度**: 1000件のデータを10秒以内に処理
- **ファイルサイズ上限**: 10MB以内を推奨

#### 9.2 互換性
- **Python**: 3.6以降
- **OS**: Windows 10以降

#### 9.3 セキュリティ
- すべての処理はローカルで完結
- ファイルデータはメモリ上でのみ処理
- ネットワーク通信なし

---

### 10. 制約事項

1. **入力ファイルのエンコーディング**: UTF-8形式必須
2. **改行コード**: CRLF, LF, CR すべて対応
3. **resourceNameフィルタ**: `people/`で始まるもののみ処理
4. **ラベルの区切り文字**: `:::` (コロン3つ)

### 11. 用語集

| 用語 | 説明 |
|------|------|
| TargetEmail | 対象GWSアカウントのメールアドレス（連絡先データのUser） |
| ContactID | Google People APIのresourceName（people/cXXXXXXXXXXXXXXXXXX） |
| PrimaryLabel | myContactsのラベルID（contactGroups/myContacts） |
| SecondaryLabel | ユーザー定義のラベルID（contactGroups/XXXX） |
| resourceName | Google APIのリソース識別子 |
| people/ | 通常の連絡先を示すプレフィックス |
| otherContacts/ | その他の連絡先を示すプレフィックス（処理対象外） |
