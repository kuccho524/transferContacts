# 1. 統合GUIアプリケーション

## 1.1 概要
**ファイル**: `transferContacts_master.py`

統合GUIアプリケーションは、全工程（Step 2〜7）を一括実行するためのメインエントリーポイント。

## 1.2 クラス設計

### 1.2.1 TransferContactsMasterGUI クラス

**責務**: 統合GUIアプリケーションのメインクラス

**属性**:
```python
class TransferContactsMasterGUI:
    # GUI要素
    root: tk.Tk                              # メインウィンドウ
    base_dir: Path                           # スクリプトのベースディレクトリ

    # ファイルパス変数（tk.StringVar）
    export_csv_path: tk.StringVar            # エクスポートCSVパス
    target_email: tk.StringVar               # ターゲットメールアドレス
    contact_csv_path: tk.StringVar           # 連絡先CSV出力先
    label_csv_path: tk.StringVar             # ラベルCSV出力先
    log_dir: tk.StringVar                    # ログ出力先（デフォルト: "./logs"）

    # 処理ステップのチェックボックス（tk.BooleanVar）
    step2_enabled: tk.BooleanVar             # Step 2: CSV抽出
    step3_enabled: tk.BooleanVar             # Step 3: 連絡先登録
    step4_enabled: tk.BooleanVar             # Step 4: ラベル作成
    step5_enabled: tk.BooleanVar             # Step 5: 登録データ取得
    step6_enabled: tk.BooleanVar             # Step 6: データ突合
    step7_enabled: tk.BooleanVar             # Step 7: ラベル登録

    # 実行時変数
    work_dir: Path                           # 作業ディレクトリ（runs/YYYYMMDD_HHMMSS）
    gitbash_path: str                        # Gitbashのパス

    # 中間ファイル保存先
    intermediate_contact_csv: Path           # 連絡先CSV（Step 2出力）
    intermediate_label_csv: Path             # ラベルCSV（Step 2出力）
    intermediate_contacts_csv: Path         # 登録済み連絡先CSV（Step 5出力）
    intermediate_contactgroups_csv: Path     # 登録済みラベルCSV（Step 5出力）
    final_labels_csv_list: List[Path]        # ラベル登録CSV（Step 6出力、複数）

    # UI要素
    log_text: scrolledtext.ScrolledText      # ログ表示エリア
    process_button: tk.Button                # 処理開始ボタン
```

## 1.3 メソッド設計

### 1.3.1 初期化メソッド

#### `__init__(self, root: tk.Tk)`
**責務**: GUIの初期化

**処理フロー**:
1. メインウィンドウ設定（タイトル: "Google Contacts 連絡先移行ツール - 統合版"、サイズ: 1000x900）
2. ベースディレクトリ取得（`Path(__file__).parent.resolve()`）
3. 変数の初期化（StringVar、BooleanVar）
4. 中間ファイル変数の初期化
5. Gitbash検出（`find_gitbash()`）
6. UI構築（`create_widgets()`）
7. 初期ログ出力（Gitbash検出結果）

### 1.3.2 Gitbash関連メソッド

#### `find_gitbash(self) -> str | None`
**責務**: Gitbash (bash.exe) のパスを検索

**検索パス**:
1. `C:\Program Files\Git\bin\bash.exe`
2. `C:\Program Files (x86)\Git\bin\bash.exe`
3. `%USERPROFILE%\AppData\Local\Programs\Git\bin\bash.exe`

**戻り値**: 見つかった場合はパス文字列、見つからない場合はNone

#### `run_gam_via_gitbash(self, gam_command: str) -> str`
**責務**: Gitbash経由でGAMコマンドを実行

**引数**:
- `gam_command`: 実行するGAMコマンド

**処理フロー**:
1. Gitbashパスの存在チェック
2. 存在しない場合は`run_gam_via_powershell_utf8()`にフォールバック
3. subprocess.run()で実行
4. タイムアウト: 600秒（10分）
5. エンコーディング: UTF-8
6. エラー発生時は例外をスロー

**戻り値**: コマンド実行結果（UTF-8文字列）

#### `run_gam_via_powershell_utf8(self, gam_command: str) -> str`
**責務**: PowerShell UTF-8版でGAMコマンドを実行（フォールバック）

**PowerShellスクリプト**:
```powershell
$OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$PSDefaultParameterValues['*:Encoding'] = 'utf8'
{gam_command}
```

### 1.3.3 処理実行メソッド

#### `confirm_and_run(self)`
**責務**: 実行確認ダイアログを表示し、処理開始

**バリデーション**:
1. エクスポートCSVが選択されているか
2. ターゲットメールアドレスが入力されているか
3. メールアドレス形式のチェック（正規表現: `^[^\s@]+@[^\s@]+\.[^\s@]+$`）

#### `run_all_steps(self)`
**責務**: 全ステップを順次実行（別スレッド起動）

**処理フロー**:
1. 処理ボタンを無効化
2. 別スレッドで`process_worker()`を起動

#### `process_worker(self)`
**責務**: 処理ワーカースレッド

**処理フロー**:
1. 作業ディレクトリ作成（`runs/YYYYMMDD_HHMMSS/`）
2. 各ステップを有効/無効に応じて実行
3. 完了ダイアログ表示
4. 処理ボタンを有効化

### 1.3.4 各ステップ実行メソッド

#### `execute_step2(self)`
**責務**: Step 2: CSV抽出

**実行コマンド**:
```bash
python csvContactsFirst/convert_contacts.py {export_csv} {contact_csv} {label_csv} --log-level INFO
```

**タイムアウト**: 300秒（5分）

**出力ファイル**:
- `intermediate_contact_csv`: 連絡先.csv
- `intermediate_label_csv`: ラベル.csv

#### `execute_step3(self)`
**責務**: Step 3: 連絡先登録

**実行コマンド**:
```powershell
powershell.exe -ExecutionPolicy Bypass -File registerContacts/run_register_cli.ps1
  -ContactCsvFile {contact_csv}
  -TargetUserEmail {target_email}
  -LogDir {log_dir}
```

**タイムアウト**: 1800秒（30分）

#### `execute_step4(self)`
**責務**: Step 4: ラベル作成（重複除去対応）

**処理フロー**:
1. ラベルCSVを読み込み（csv.DictReader）
2. すべての列からラベル名を収集
3. `set()`で重複を除去
4. `* myContacts`、`myContacts`を除外
5. `* `プレフィックスを削除
6. ソートして各ラベルに対してGAMコマンド実行

**GAMコマンド**:
```bash
gam user {target_email} create contactgroup name "{label_name}"
```

**エラー処理**:
- `already exists`を検出した場合はスキップカウント

#### `execute_step5(self)`
**責務**: Step 5: 登録データ取得

**GAMコマンド**:
```bash
gam user {target_email} print contacts
gam user {target_email} print contactgroups
```

**出力ファイル**:
- `intermediate_contacts_csv`: contacts.csv
- `intermediate_contactgroups_csv`: contactgroups.csv

#### `execute_step6(self)`
**責務**: Step 6: データ突合

**実行コマンド**:
```bash
python csvContactsSecond/main.py
  --export-data {export_csv}
  --registered-data {contact_csv}
  --label-data {contactgroups_csv}
  --contacts-data {contacts_csv}
  --target-email {target_email}
  --output {output_base}.csv
```

**タイムアウト**: 300秒（5分）

**出力ファイル検出**:
- globパターン: `contacts_labels_*.csv`
- `final_labels_csv_list`に保存

#### `execute_step7(self)`
**責務**: Step 7: ラベル登録（複数CSVファイル対応）

**処理フロー**:
1. `final_labels_csv_list`から各CSVファイルを処理
2. ヘッダー行からラベル列を自動検出
3. contactgroupパラメータを動的に構築
4. GAMコマンド実行

**ラベル列検出ルール**:
- `*Label`で終わる列名
- `TargetEmail`、`ContactID`を除外

**GAMコマンド**:
```bash
gam csv "{csv_file}" gam user "~TargetEmail" update contacts "~ContactID"
  contactgroup "~PrimaryLabel" contactgroup "~SecondaryLabel" ...
```

# 2. CSV抽出モジュール

## 2.1 概要
**ファイル**: `csvContactsFirst/convert_contacts.py`

Google Contactsエクスポートデータから連絡先CSVとラベルCSVを生成する。

## 2.2 関数設計

### 2.2.1 `setup_logging(log_level: str) -> logging.Logger`
**責務**: ログ設定を初期化

**処理フロー**:
1. logsディレクトリを作成
2. ログファイル名を生成（`convert_YYYYMMDD_HHMMSS.log`）
3. ファイルハンドラ（UTF-8）とコンソールハンドラを設定
4. ログフォーマット: `%(asctime)s - %(levelname)s - %(message)s`

### 2.2.2 `select_files() -> tuple | None`
**責務**: ファイル選択ダイアログを表示

**戻り値**: `(input_file, contact_output, label_output)` またはキャンセル時はNone

### 2.2.3 `read_export_data(input_file: str, logger) -> list`
**責務**: エクスポートデータを読み込む

**処理**: CSV読み込み（UTF-8）→ 辞書リストで返却

### 2.2.4 `get_ordinal_suffix(num: int) -> str`
**責務**: 序数の接尾辞を取得

**マッピング**:
| num | 戻り値 |
|-----|--------|
| 1 | Primary |
| 2 | Secondary |
| 3 | Third |
| 4 | Fourth |
| 5 | Fifthry |
| ... | ... |

### 2.2.5 `extract_emails_and_phones(row: dict) -> tuple`
**責務**: レコードからメールアドレスと電話番号を抽出

**処理**:
- `E-mail N - Label`、`E-mail N - Value` を検出
- `Phone N - Label`、`Phone N - Value` を検出
- ` ::: ` で区切られた複数値を分割

**戻り値**: `(emails: list, phones: list)`

### 2.2.6 `generate_contact_csv(data: list, output_file: str, logger)`
**責務**: 連絡先登録用CSVを生成

**出力列**:
- 基本情報: FirstName, LastName, HiraganaFirstName, HiraganaLastName, OrganizationName, OrganizationDepartment
- メール: PrimaryEmailLabel, PrimaryEmailAddress, SecondaryEmailLabel, SecondaryEmailAddress, ...
- 電話: PrimaryPhoneLabel, PrimaryPhoneNumber, SecondaryPhoneLabel, SecondaryPhoneNumber, ...

**列数**: データ内の最大メール数・電話数に応じて動的に決定

### 2.2.7 `parse_labels(labels_str: str) -> list`
**責務**: ラベル文字列を分割

**処理**:
- `:::` で分割（スペースの有無に関係なく対応）
- トリムして空でない要素のみ返す

### 2.2.8 `get_label_column_name(index: int) -> str`
**責務**: ラベル列の列名を取得

**マッピング**:
| index | 戻り値 |
|-------|--------|
| 0 | PrimaryLabel |
| 1 | SecondaryLabel |
| 2 | ThirdLabel |
| 3 | FourthLabel |
| ... | ... |
| 21 | TwentySecondLabel |
| 22以降 | 23rdLabel, 24thLabel, ... |

### 2.2.9 `generate_label_csv(data: list, output_file: str, logger)`
**責務**: ラベル作成用CSVを生成

**処理フロー**:
1. 全レコードからラベルを抽出
2. `* myContacts`、`myContacts`を除外
3. 最大Secondary列数を計算
4. ヘッダー行を動的生成
5. 各行のPrimaryLabelは`* myContacts`固定

# 3. データ突合モジュール

## 3.1 概要
**ディレクトリ**: `csvContactsSecond/`

エクスポートデータと登録データを突合し、ラベル登録用CSVを生成する。

## 3.2 モジュール構成

### 3.2.1 csv_parser.py

#### `parse_csv(file_path: str) -> List[Dict[str, str]]`
**責務**: CSVファイルを読み込んでパース

**処理フロー**:
1. UTF-8で読み込み
2. BOM付きUTF-8の場合は除去（`\ufeff`）
3. エンコーディング検証（`validate_encoding()`）
4. csv.DictReaderでパース

**エラー処理**:
- FileNotFoundError: ファイル不存在
- UnicodeDecodeError: UTF-8以外
- ValueError: CSV形式不正

#### `validate_encoding(text: str) -> bool`
**責務**: エンコーディングを検証

**チェック項目**:
- 置換文字（U+FFFD）の検出
- 連続する置換文字（`��`）の検出
- 制御文字（タブ、改行以外）の検出

### 3.2.2 data_processor.py

#### `extract_export_data(export_data: List[Dict]) -> List[Dict]`
**責務**: エクスポートデータから必要な列を抽出

**抽出列**:
| 入力列 | 出力キー |
|--------|----------|
| First Name | first_name |
| Last Name | last_name |
| E-mail 1 - Value | email |
| Labels | labels |

#### `extract_contacts_data(contacts_data: List[Dict]) -> List[Dict]`
**責務**: 連絡先データから必要な列を抽出（`people/`のみ）

**抽出列**:
| 入力列 | 出力キー |
|--------|----------|
| User | user |
| names.0.givenName | first_name |
| names.0.familyName | last_name |
| emailAddresses.0.value | email |
| resourceName | resource_name |

**フィルタ**: `resourceName.startswith('people/')` のみ処理

#### `extract_labels_data(labels_data: List[Dict]) -> List[Dict]`
**責務**: ラベルデータから必要な列を抽出

**抽出列**:
| 入力列 | 出力キー |
|--------|----------|
| User | user |
| resourceName | resource_name |
| name | name |

#### `match_data(export_data, contacts_data) -> Tuple[List[Dict], List[Dict]]`
**責務**: エクスポートデータと連絡先データを突合

**一致条件**:
- 姓（last_name）が一致
- 名（first_name）が一致
- メールアドレス（email、小文字で比較）が一致

**戻り値**: `(matched_data, skip_list)`

#### `parse_labels(labels_str: str) -> List[str]`
**責務**: Labels列をパース

**処理**: `:::` で分割、トリム、空文字列除外

#### `create_label_map(labels_data: List[Dict]) -> Dict[str, str]`
**責務**: ラベル名からラベルIDへのマッピングを作成

**戻り値**: `{ラベル名: ラベルID}` の辞書

#### `classify_labels(labels_list: List[str], label_map: Dict) -> Dict`
**責務**: ラベルをPrimaryとSecondaryに分類

**処理**:
- `* `プレフィックスを削除
- `myContacts` → PrimaryLabel
- その他 → SecondaryLabel以降

**戻り値**:
```python
{
    'primary_label': str,        # contactGroups/myContacts
    'secondary_labels': List[str] # ラベルIDのリスト（個数制限なし）
}
```

#### `get_ordinal_adjective(num: int) -> str`
**責務**: 数字から序数形容詞を取得

**マッピング（num: 序数形容詞）**:
| num | 戻り値 | 説明 |
|-----|--------|------|
| 2 | Third | SecondaryLabelの次 |
| 3 | Fourth | |
| ... | ... | |
| 49 | Fiftieth | |
| 50以降 | 51st, 52nd, 53rd, 54th... | 数値表記 |

#### `transform_to_output(matched_data, label_map) -> Dict`
**責務**: マッチングデータを出力形式に変換

**処理フロー**:
1. 各マッチングデータに対してラベル解析
2. 出力行を作成（TargetEmail, ContactID, PrimaryLabel, SecondaryLabel...）
3. 最大ラベル数を記録

**戻り値**:
```python
{
    'data': List[Dict],      # 出力データ
    'max_label_count': int   # 最大ラベル数
}
```

#### `generate_output_columns(max_label_count: int) -> List[str]`
**責務**: 出力CSV列を動的に生成

**列名**: `['TargetEmail', 'ContactID', 'PrimaryLabel', 'SecondaryLabel', 'ThirdLabel', ...]`

#### `transform_to_output_data(...)` -> Dict
**責務**: エクスポートデータを出力形式に変換（ラベル数ごとにグループ化）

**戻り値**:
```python
{
    'grouped_data': Dict[int, List[Dict]],  # ラベル数 -> データリスト
    'max_label_count': int
}
```

### 3.2.3 csv_generator.py

#### `generate_csv(data: List[Dict], columns: List[str]) -> str`
**責務**: データからCSV文字列を生成

**処理**:
- ヘッダー行を追加
- データ行を追加
- LF改行で結合

#### `escape_csv_field(value: str) -> str`
**責務**: CSVフィールドをエスケープ

**エスケープルール**:
- ダブルクォート、カンマ、改行が含まれる場合はダブルクォートで囲む
- ダブルクォートは2つに（`"` → `""`）

#### `save_csv(csv_content: str, output_path: str)`
**責務**: CSV文字列をファイルに保存

**形式**: UTF-8、BOMなし、LF改行

#### `generate_filename(prefix: str = 'contacts_labels') -> str`
**責務**: タイムスタンプ付きファイル名を生成

**形式**: `{prefix}_YYYYMMDD_HHMMSS.csv`

#### `generate_skip_list_csv(skip_list: List[Dict]) -> str`
**責務**: スキップリストのCSV文字列を生成

**列**: 姓, 名, メールアドレス, 理由

#### `save_skip_list_csv(skip_list: List[Dict], output_path: str)`
**責務**: スキップリストをCSVファイルに保存

# 4. 連絡先登録モジュール

## 4.1 概要
**ディレクトリ**: `registerContacts/`

GAMコマンドを使用して連絡先とラベルを登録するPowerShellスクリプト。

## 4.2 run_register_cli.ps1

### 4.2.1 パラメータ

| パラメータ | 必須 | デフォルト | 説明 |
|------------|------|------------|------|
| ContactCsvFile | ✓ | - | 連絡先CSVファイルパス |
| LabelCsvFile | - | "" | ラベルCSVファイルパス |
| TargetUserEmail | ✓ | - | ターゲットユーザーのメールアドレス |
| TempJsonFile | - | $env:TEMP\contact_temp.json | 一時JSONファイルパス |
| LogDir | - | "./logs" | ログ保存先ディレクトリ |

### 4.2.2 処理フロー

#### Phase 1: 連絡先の登録
1. 連絡先CSVを読み込み（`Import-Csv -Encoding UTF8`）
2. 各レコードに対してループ処理:
   - JSON構築（names, organizations, emailAddresses, phoneNumbers）
   - 一時JSONファイルに保存
   - GAMコマンド実行: `gam user {email} create contact json file {json}`
3. 一時ファイル削除

#### Phase 2: ラベルの一括作成（任意）
1. ラベルCSVのヘッダー行を読み取り
2. ラベル列を自動検出（`*Label`で終わる列、PrimaryLabel除外）
3. 各ラベル列に対してGAMコマンド実行:
   `gam csv {csv} gam user {email} create contactgroup name "~{column}"`

### 4.2.3 JSON構造

```json
{
    "names": [{
        "givenName": "ユーザー",
        "familyName": "サンプル",
        "phoneticGivenName": "ゆーざー",
        "phoneticFamilyName": "さんぷる"
    }],
    "organizations": [{
        "name": "Example Corp",
        "department": "Operations",
        "type": "work"
    }],
    "emailAddresses": [
        { "value": "sample.user@example.com", "type": "work", "metadata": { "primary": true } },
        { "value": "sample.user.alt@example.com", "type": "home" }
    ],
    "phoneNumbers": [
        { "value": "090-0000-0001", "type": "mobile", "metadata": { "primary": true } },
        { "value": "03-0000-0002", "type": "work" }
    ]
}
```
