本書は以下の現行実装を対象とする。

- `csvContactsSecond/main.py`

## 2. 統合 GUI 設計

### 2.1 クラス

#### `TransferContactsMasterGUI`

主な属性:

| 属性 | 役割 |
|------|------|
| `base_dir` | スクリプト基準ディレクトリ |
| `python_executable` | 子プロセス呼び出しに使う Python 実体 |
| `gam_executable` | `shutil.which("gam")` で検出した GAM 実体 |
| `gam_wrapper_script` | `registerContacts/invoke_gam.ps1` のパス |
| `log_queue` | ワーカースレッドから GUI へログを渡すキュー |
| `export_csv_path` | エクスポート CSV の入力パス |
| `target_email` | ターゲットメールアドレス |
| `contact_csv_path` | 連絡先 CSV 出力先または既存ファイル |
| `label_csv_path` | ラベル CSV 出力先または既存ファイル |
| `log_dir` | PowerShell ログ出力先 |
| `step2_enabled`〜`step7_enabled` | 各ステップの有効 / 無効 |
| `work_dir` | 実行ごとの作業ディレクトリ |
| `intermediate_contact_csv` | Step 2 出力または既存の連絡先 CSV |
| `intermediate_label_csv` | Step 2 出力または既存のラベル CSV |
| `intermediate_contacts_csv` | Step 5 出力の `contacts.csv` |
| `intermediate_contactgroups_csv` | Step 5 出力の `contactgroups.csv` |
| `final_labels_csv_list` | Step 6 出力の `contacts_labels_*.csv` 一覧 |

### 2.2 初期化

`__init__()` の処理:
1. 画面サイズとタイトルを設定
2. `base_dir`, `python_executable`, `gam_executable`, `gam_wrapper_script` を初期化
3. Tk 変数と中間ファイル参照を初期化
4. UI を構築
5. `root.after(100, self.flush_log_queue)` でログ反映ループを開始
6. Python 実行環境と GAM 検出結果をログ出力

### 2.3 ログ反映方式

Tkinter のスレッド制約に対応するため、ログ表示は以下で分離する。

- `append_log_line()`
  GUI スレッドでウィジェットに 1 行追記
- `flush_log_queue()`
  `queue.Queue` からログを取り出して UI に反映
- `log()`
  メインスレッドなら直接反映、ワーカースレッドならキューへ投入

### 2.4 実行前バリデーション

#### `requires_gam()`
Step 3〜7 のいずれかが選択されているか判定する。

#### `existing_input_path(value)`
既存ファイルとして利用可能なパスのみ返す。

#### `validate_step_dependencies()`
以下を検証する。

- Step 3〜7 実行時に `gam` が利用可能か
- Step 3 単独実行時に既存の連絡先 CSV があるか
- Step 4 単独実行時に既存のラベル CSV があるか
- Step 6 実行時に Step 5 が有効か
- Step 6 実行時に連絡先 CSV が確保できるか
- Step 7 実行時に Step 6 が有効か

#### `prepare_intermediate_inputs()`
Step 2 を実行しない場合、ユーザー指定の既存 CSV を中間入力として取り込む。

### 2.5 実行フロー

#### `confirm_and_run()`
1. 必須入力の有無を確認
2. メール形式を正規表現で検証
3. `validate_step_dependencies()` を実行
4. 確認ダイアログを表示
5. `run_all_steps()` を起動

#### `run_all_steps()`
- 処理開始ボタンを無効化
- `process_worker()` を別スレッドで開始

#### `process_worker()`
1. `prepare_intermediate_inputs()` を実行
2. `runs/YYYYMMDD_HHMMSS/` を作成
3. 有効な Step を順に実行
4. 完了時は情報ダイアログ、失敗時はエラーダイアログを表示
5. 最後にボタンを再度有効化

## 3. GAM 実行設計

### 3.1 `run_gam(gam_args, timeout=600)`

目的:
- GAM コマンドを PowerShell ラッパー経由で安全に実行する

処理:
1. `invoke_gam.ps1` の存在確認
2. 引数配列 `gam_args` を JSON 化
3. `powershell.exe -File invoke_gam.ps1 -GamArgsJson ...` を実行
4. 標準出力 / 標準エラーを UTF-8 として収集
5. `(returncode, combined_output)` を返す

### 3.2 `invoke_gam.ps1`

役割:
- `gam` を UTF-8 で呼び出す薄いラッパー

処理:
1. `gam` の存在確認
2. `GamArgsJson` を `ConvertFrom-Json` で配列化
3. `& gam @GamArgs` で実行
4. 出力を `Out-String` で標準出力へ返す
5. 終了コードをそのまま返す

## 4. ステップ別詳細設計

### 4.1 Step 2: `execute_step2()`

実行コマンド:

```bash
{sys.executable} csvContactsFirst/convert_contacts.py \
  {export_csv} {contact_csv} {label_csv} --log-level INFO
```

仕様:
- 出力先が未指定なら `work_dir/連絡先.csv`, `work_dir/ラベル.csv`
- 成功時は `intermediate_contact_csv`, `intermediate_label_csv` に保存
- タイムアウトは 300 秒

### 4.2 Step 3: `execute_step3()`

実行コマンド:

```powershell
powershell.exe -ExecutionPolicy Bypass -File registerContacts/run_register_cli.ps1 `
  -ContactCsvFile {intermediate_contact_csv} `
  -TargetUserEmail {target_email} `
  -LogDir {log_dir}
```

仕様:
- タイムアウトは 1800 秒
- 標準出力と標準エラーを GUI ログへ反映

### 4.3 Step 4: `execute_step4()`

処理:
1. ラベル CSV を `csv.DictReader` で読む
2. 全列からラベル名を収集
3. `* myContacts` / `myContacts` を除外
4. `* ` プレフィックスを除去
5. 重複を除去してソート
6. `run_gam(["user", target, "create", "contactgroup", "name", label])` を実行

例外:
- 既存ラベルはスキップ
- 一部失敗時も Step 全体は継続可能

### 4.4 Step 5: `execute_step5()`

処理:
- `gam user {target} print contacts`
- `gam user {target} print contactgroups`

出力:
- `work_dir/contacts.csv`
- `work_dir/contactgroups.csv`

成功時:
- `intermediate_contacts_csv`
- `intermediate_contactgroups_csv`
  に保存

### 4.5 Step 6: `execute_step6()`

実行コマンド:

```bash
{sys.executable} csvContactsSecond/main.py \
  --export-data {export_csv} \
  --registered-data {intermediate_contact_csv} \
  --label-data {intermediate_contactgroups_csv} \
  --contacts-data {intermediate_contacts_csv} \
  --target-email {target_email} \
  --output {work_dir}/contacts_labels.csv
```

仕様:
- 実際の出力は `contacts_labels_1.csv`, `contacts_labels_2.csv` のように分割される
- `glob("contacts_labels_*.csv")` で検出し、`final_labels_csv_list` に格納
- タイムアウトは 300 秒

### 4.6 Step 7: `execute_step7()`

処理:
1. `final_labels_csv_list` を順に処理
2. ヘッダーから `*Label` 列を検出
3. `TargetEmail`, `ContactID` は除外
4. `gam csv {file} gam user "~TargetEmail" update contacts "~ContactID" ...` の引数配列を生成
5. `run_gam()` で実行

出力:
- 成功 / 失敗件数をログに集計

## 5. CSV 抽出モジュール設計

対象: `csvContactsFirst/convert_contacts.py`

### 5.1 主要関数

| 関数 | 役割 |
|------|------|
| `setup_logging()` | `logs/convert_*.log` を生成してロガーを返す |
| `select_files()` | GUI ダイアログで入力 / 出力ファイルを選択する |
| `read_export_data()` | エクスポート CSV を UTF-8 で読み込む |
| `get_ordinal_suffix()` | `Primary`, `Secondary`, `Third`, `Fourth` などを返す |
| `extract_emails_and_phones()` | `E-mail N`, `Phone N` を可変件数で抽出する |
| `generate_contact_csv()` | 連絡先登録用 CSV を生成する |
| `parse_labels()` | `Labels` を `:::` 区切りで分解する |
| `get_label_column_name()` | `PrimaryLabel`, `SecondaryLabel`, `ThirdLabel` などを返す |
| `generate_label_csv()` | ラベル作成用 CSV を生成する |

### 5.2 重要仕様

- `extract_emails_and_phones()` は `" ::: "` で複数値を展開する
- 連絡先 CSV のメール列 / 電話列は最大件数に応じて動的生成される
- ラベル CSV の列数もデータ件数に応じて動的生成される
- `generate_label_csv()` は `PrimaryLabel` に常に `* myContacts` を設定する

## 6. データ突合モジュール設計

対象: `csvContactsSecond/main.py`, `csvContactsSecond/data_processor.py`

### 6.1 CLI フロー

`main.py` の処理順:
1. 引数を検証
2. 4 種類の CSV を読み込む
3. `create_label_map()` と `create_contact_map()` を生成
4. `check_consistency()` で登録用 CSV との差異を確認
5. `transform_to_output_data()` で出力データへ変換
6. ラベル数ごとに `contacts_labels_*.csv` を保存
7. 必要なら `skip_list_*.csv` も保存

### 6.2 `data_processor.py` の主要関数

| 関数 | 役割 |
|------|------|
| `normalize_email()` | メールアドレスの比較用正規化 |
| `unique_preserve_order()` | 順序を保った重複除去 |
| `extract_export_emails()` | エクスポート CSV から全メール抽出 |
| `extract_registered_emails()` | 登録用 CSV から全メール抽出 |
| `extract_contact_emails()` | `contacts.csv` から全メール抽出 |
| `build_identity_keys()` | 氏名 + メールの照合キー生成 |
| `find_matching_contact_candidates()` | 候補連絡先の検索 |
| `extract_export_data()` | エクスポート元データの抽出 |
| `extract_registered_data()` | 登録用 CSV の抽出 |
| `extract_contacts_data()` | `people/` 連絡先のみ抽出 |
| `extract_labels_data()` | ラベル一覧抽出 |
| `create_label_map()` | ラベル名 → ラベル ID の辞書を作成 |
| `create_contact_map()` | 氏名 + メール群で照合できる連絡先マップを作成 |
| `check_consistency()` | 登録用 CSV との不一致を検出 |
| `transform_to_output_data()` | ラベル登録用データへ変換し、ラベル数ごとにグループ化 |
| `generate_output_columns()` | ラベル数に応じた列名を生成 |

### 6.3 照合ルール

- 連絡先候補は「姓 + 名 + 任意メール」で照合
- 1 件の連絡先に複数メールアドレスがあっても対応
- `contacts.csv` では `resourceName` が `people/` で始まる行のみ対象
- 氏名が空のデータは、メール一致かつ候補が一意の場合のみ採用

### 6.4 整合性チェック

`check_consistency()` の返却値:

```python
{
    "skip_list": [...],
    "skip_row_indexes": {...}
}
```

用途:
- `skip_list`: CSV 出力用の不一致一覧
- `skip_row_indexes`: Step 6 で処理対象外にする元データ行

### 6.5 出力グループ化

`transform_to_output_data()` は以下を返す。

```python
{
    "grouped_data": {2: [...], 3: [...]},
    "max_label_count": 3
}
```

`main.py` は `grouped_data` を使い、ラベル数ごとに別ファイルを生成する。

## 7. 連絡先登録モジュール設計

対象: `registerContacts/run_register_cli.ps1`

### 7.1 パラメータ

| パラメータ | 必須 | 説明 |
|------------|------|------|
| `ContactCsvFile` | ✓ | 連絡先 CSV |
| `LabelCsvFile` | - | ラベル CSV |
| `TargetUserEmail` | ✓ | 登録先アカウント |
| `TempJsonFile` | - | 一時 JSON ファイル |
| `LogDir` | - | 互換用のログディレクトリ引数。現行実装では存在確認と作成のみを行う |

### 7.2 主要関数

| 関数 | 役割 |
|------|------|
| `Log-Message` | タイムスタンプ付きログ出力 |
| `Invoke-GamCommand` | 引数配列で GAM を実行 |
| `Get-DefaultMethodType` | 列名に応じた既定 type を返す |
| `Get-EmailObjects` | `*EmailAddress` 列を JSON 形式へ変換 |
| `Get-PhoneObjects` | `*PhoneNumber` 列を JSON 形式へ変換 |
| `Get-LabelColumns` | ラベル CSV から対象列を検出 |

### 7.3 連絡先登録処理

処理:
1. 連絡先 CSV を UTF-8 で読み込む
2. 各レコードから `names`, `organizations`, `emailAddresses`, `phoneNumbers` を構築
3. `TempJsonFile` に書き出す
4. `gam user {TargetUserEmail} create contact json file {TempJsonFile}` を実行
5. 最後に一時 JSON を削除する

### 7.4 ラベル作成処理

`LabelCsvFile` が指定されている場合のみ実行する。

処理:
1. `Get-LabelColumns()` で `*Label` 列を抽出
2. `PrimaryLabel`, `TargetEmail`, `ContactID` は除外
3. 各列に対して `gam csv ... create contactgroup name "~Column"` を実行

## 8. 共通仕様

### 8.1 ファイルエンコーディング
- 入力: UTF-8
- 出力: UTF-8
- 改行: 主に LF

### 8.2 一時ファイルとログ
- Step 2 の Python ログ:
  `logs/convert_YYYYMMDD_HHMMSS.log`
- 統合 GUI の作業ディレクトリ:
  `runs/YYYYMMDD_HHMMSS/`
- `run_register_cli.ps1` の `LogDir`:
  現行実装ではディレクトリを作成するが、専用ログファイルは出力しない

### 8.3 エラー処理
- Python 側は例外時に標準出力 / 標準エラーを GUI に転記
- PowerShell 側は終了コード 1 で失敗を返す
- GUI 側はメッセージボックスで完了 / エラーを通知
