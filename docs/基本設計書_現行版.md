# Google Contacts 連絡先移行ツール - 現行版 基本設計書

> 注記: この文書は単体ツール中心だった時期の設計書です。公開版の推奨導線は `transferContacts_master.py` と `csvContactsSecond/main.py` で、旧単体 GUI / 旧補助スクリプトは公開対象に含めません。

## 1. システム概要

### 1.1 システム名
Google Contacts 連絡先移行ツール

### 1.2 目的
旧Googleアカウントから新アカウントへ連絡先を移行する際の全工程（手順2〜7）を、1つのGUIアプリケーションで一気通貫で実行できるようにする。

### 1.3 対象ユーザー
企業の社内SE/情シス担当者

### 1.4 開発言語・環境
- **言語**: Python 3.6以降、PowerShell 5.1以降
- **GUI**: tkinter（Python標準ライブラリ）
- **外部コマンド**: GAM（Google Apps Manager）
- **シェル**: Gitbash（文字化け対策）、PowerShell（フォールバック）
- **実行環境**: Windows 10以降（MINGW64）

### 1.5 背景・課題
現状、連絡先移行には以下の問題がある：
- 各工程（手順2〜7）を個別に手動実行する必要がある
- 実行環境が混在（Python、PowerShell、Gitbash）
- 各工程間のファイル受け渡しが手作業
- GAMコマンドの文字化け問題（特に日本語ラベル名）

### 1.6 解決アプローチ
- **統合GUIアプリケーション**で全工程を自動化
- **Gitbash経由でGAMコマンド実行**により文字化けを回避
- **任意のファイル選択**と**実行前の確認ダイアログ**でユーザビリティ向上
- **実行ログのリアルタイム表示**で進捗を可視化
- **ラベル数ごとのCSV分割出力**でGAMコマンドの柔軟な実行をサポート

### 3.2 処理ステップ詳細

#### Step 2: CSV抽出（F-004）
- **入力**: Google Contactsエクスポートデータ（CSV）
- **出力**: 連絡先CSV、ラベルCSV
- **処理内容**:
  - エクスポートCSVを読み込み
  - 連絡先情報（氏名、メール、電話等）を抽出して連絡先CSV生成
  - ラベル情報を抽出してラベルCSV生成
  - メール・電話の複数値（`:::`区切り）を個別列に展開
  - ラベル列を動的に生成（SecondaryLabel, ThirdLabel...）

#### Step 3: 連絡先登録（F-005）
- **入力**: 連絡先CSV
- **出力**: なし（Google Contactsに直接登録）
- **処理内容**:
  - 連絡先CSVをレコードごとに処理
  - JSON形式に変換
  - GAMコマンド `gam user xxx create contact json file xxx.json` で登録
  - 組織情報、メールアドレス、電話番号を含む

#### Step 4: ラベル作成（F-006）
- **入力**: ラベルCSV
- **出力**: なし（Google Contactsに直接作成）
- **処理内容**:
  - ラベルCSVから全ラベル名を収集
  - `set()`で重複を除去
  - `* myContacts`およびシステムラベルを除外
  - `* `プレフィックスを削除
  - GAMコマンド `gam user xxx create contactgroup name "ラベル名"` で作成
  - 既存ラベルはスキップ（エラーログ検出）

#### Step 5: 登録データ取得（F-007）
- **入力**: ターゲットメールアドレス
- **出力**: contacts.csv、contactgroups.csv
- **処理内容**:
  - GAMコマンド `gam user xxx print contacts` で連絡先一覧取得
  - GAMコマンド `gam user xxx print contactgroups` でラベル一覧取得
  - UTF-8でファイル保存

#### Step 6: データ突合（F-008）
- **入力**: エクスポートCSV、連絡先CSV、contacts.csv、contactgroups.csv
- **出力**: contacts_labels_N.csv（ラベル数ごとに分割）
- **処理内容**:
  - エクスポートデータと登録データを姓名・メールアドレスで突合
  - 一致したデータにラベルIDをマッピング
  - ラベル数ごとにCSVファイルを分割出力
  - スキップリストCSVも生成

#### Step 7: ラベル登録（F-009）
- **入力**: contacts_labels_N.csv（複数）
- **出力**: なし（Google Contactsに直接登録）
- **処理内容**:
  - 各CSVファイルのヘッダーからラベル列を自動検出
  - contactgroupパラメータを動的に構築
  - GAMコマンド `gam csv xxx.csv gam user "~TargetEmail" update contacts "~ContactID" contactgroup "~PrimaryLabel" contactgroup "~SecondaryLabel"...` で登録

```csv
First Name,Last Name,E-mail 1 - Value,Phone 1 - Value,Labels,...
サンプル,ユーザー,sample.user@example.com,090-0000-0001,Team A ::: * myContacts,...
```

#### 4.2.2 連絡先CSV（中間ファイル）
```csv
FirstName,LastName,HiraganaFirstName,HiraganaLastName,OrganizationName,OrganizationDepartment,PrimaryEmailLabel,PrimaryEmailAddress,SecondaryEmailLabel,SecondaryEmailAddress,PrimaryPhoneLabel,PrimaryPhoneNumber,...
ユーザー,サンプル,ゆーざー,さんぷる,Example Corp,Operations,work,sample.user@example.com,,,mobile,090-0000-0001,...
```

#### 4.2.3 ラベルCSV（中間ファイル）
```csv
PrimaryLabel,SecondaryLabel,ThirdLabel
* myContacts,Team A,
* myContacts,Team B,Operations
```

#### 4.2.4 contacts.csv（GAM出力）
```csv
User,resourceName,names.0.givenName,names.0.familyName,emailAddresses.0.value,...
user@example.com,people/c1234567890,ユーザー,サンプル,sample.user@example.com,...
```

#### 4.2.5 contactgroups.csv（GAM出力）
```csv
User,resourceName,name
user@example.com,contactGroups/myContacts,myContacts
user@example.com,contactGroups/abc123,Team A
```

#### 4.2.6 contacts_labels_N.csv（最終出力）
```csv
TargetEmail,ContactID,PrimaryLabel,SecondaryLabel
user@example.com,people/c1234567890,contactGroups/myContacts,contactGroups/abc123
```

## 5. 非機能要件

### 5.1 パフォーマンス
- **処理速度**: 100件の連絡先を10分以内に処理
- **応答性**: GUIの応答速度は1秒以内
- **タイムアウト**:
  - Python処理: 5分
  - PowerShell処理: 30分
  - GAMコマンド: 10分

### 5.2 ユーザビリティ
- **直感的なUI**: 日本語ラベル、明確なグループ化
- **確認ダイアログ**: 誤操作防止のための念押し確認
- **ログの可読性**: タイムスタンプ、絵文字アイコン
- **ステップ選択**: 特定ステップのみ実行可能

### 5.3 互換性
| 項目 | 要件 |
|------|------|
| Python | 3.6以降 |
| OS | Windows 10以降 |
| GAM | 最新版（UTF-8出力対応） |
| Gitbash | Git for Windows 2.x以降 |
| PowerShell | 5.1以降 |

### 5.4 セキュリティ
- すべての処理はローカルで完結
- ファイルデータはメモリ上でのみ処理
- ネットワーク通信はGAMコマンドのみ
- 中間ファイルは実行ディレクトリ内に保存
- 一時JSONファイルは処理後に自動削除

### 5.5 保守性
- **モジュール化**: 各処理を独立したモジュールに分離
- **ログ保存**: 実行ログをファイルに保存（デバッグ用）
- **バージョン管理**: Git管理、変更履歴の記録
- **日本語コメント**: すべてのコメントは日本語で記述

### 5.6 拡張性
- **処理ステップの追加**: 新しいステップを容易に追加可能
- **動的列生成**: ラベル数に応じて自動的に列を追加

## 8. 制約事項

### 8.1 技術的制約
- Python 3.6以降が必要
- Gitbash推奨（文字化け対策）
- GAMがインストール済みであること
- Windows環境でのみ動作確認

### 8.2 機能的制約
- 手順1（旧アカウントからのエクスポート）は手動実行
- Google Workspace（旧G Suite）アカウント必須
- GAMの認証設定が事前に完了していること
- ラベル名は50個まで対応（51個以降は数値表記）

### 8.3 運用的制約
- 大量データ（1000件以上）の処理は未検証
- API制限によりエラーが発生する可能性あり
- 実行中はGUI操作不可（別スレッド実行）
