# Google Contacts 連絡先移行ツール - 現行版 基本設計書

## 1. システム概要

### 1.1 システム名
Google Contacts 連絡先移行ツール

### 1.2 目的
Google Contacts のエクスポート CSV をもとに、連絡先登録用 CSV の生成、GAM を使った登録、登録後データの取得、ラベル付与用 CSV の生成、ラベル登録までを一連で実行できるようにする。

### 1.3 対象ユーザー
- Google Workspace の連絡先移行を担当する情シス担当者
- Windows 環境で GAM を使って連絡先移行を行う運用担当者

### 1.4 前提環境
- Windows 10 以降
- Python 3.6 以降
- PowerShell 5.1 以降
- `gam` コマンドが PATH に通っていること
- Google Workspace 側で必要な権限と GAM 認証設定が完了していること

### 1.5 解決したい課題
- 連絡先移行の各工程を個別に手動実行する負荷を下げる
- CSV 生成、登録、突合、ラベル付与の流れを一つの GUI から実行できるようにする
- 中間ファイルの受け渡しを実行時ディレクトリで管理し、運用手順を単純化する
- ラベル数やメール件数が可変でも処理できるようにする

## 3. 機能要件

### 3.1 主要機能一覧

| 機能ID | 機能名 | 説明 |
|--------|--------|------|
| F-001 | 入力指定機能 | エクスポート CSV とターゲットメールアドレスを指定する |
| F-002 | 出力先指定機能 | 連絡先 CSV、ラベル CSV、ログ出力先を任意指定できる |
| F-003 | ステップ選択機能 | Step 2〜7 の各工程を選択して実行できる |
| F-004 | 実行前確認機能 | 実行内容を確認ダイアログで提示する |
| F-005 | CSV 抽出機能 | エクスポート CSV から連絡先登録用 CSV とラベル CSV を生成する |
| F-006 | 連絡先登録機能 | 登録用 CSV をもとに GAM で連絡先を登録する |
| F-007 | ラベル作成機能 | ラベル CSV をもとに不足ラベルを作成する |
| F-008 | 登録データ取得機能 | `contacts.csv` と `contactgroups.csv` を取得する |
| F-009 | データ突合機能 | 元データと登録済みデータを照合し、ラベル付与用 CSV を生成する |
| F-010 | ラベル登録機能 | ラベル付与用 CSV をもとに連絡先へラベルを登録する |
| F-011 | ログ表示機能 | GUI 上に時系列ログを表示する |
| F-012 | 依存関係チェック機能 | 実行ステップの前提条件を事前に検証する |

### 3.2 ステップ別要件

#### Step 2: CSV 抽出
- 入力: Google Contacts エクスポート CSV
- 出力: 連絡先登録用 CSV、ラベル作成用 CSV
- 特徴:
  - メールアドレスと電話番号は可変列に対応
  - `:::` 区切りの複数値を展開
  - ラベル列は件数に応じて動的生成

#### Step 3: 連絡先登録
- 入力: 連絡先登録用 CSV
- 出力: Google Contacts への登録
- 特徴:
  - PowerShell で JSON を生成して GAM に渡す
  - メールアドレス / 電話番号は `*EmailAddress`, `*PhoneNumber` 列を動的に走査
  - `run_register_cli.ps1` 自体は `LabelCsvFile` 指定時のラベル作成にも対応する
  - 現行の統合 GUI ではラベル作成は Step 4 として分離して実行する

#### Step 4: ラベル作成
- 入力: ラベル作成用 CSV
- 出力: Google Contacts 上の連絡先グループ
- 特徴:
  - 重複ラベルを除去
  - `* myContacts` は除外
  - `already exists` はスキップ扱い

#### Step 5: 登録データ取得
- 入力: ターゲットメールアドレス
- 出力: `contacts.csv`, `contactgroups.csv`
- 特徴:
  - `invoke_gam.ps1` 経由で UTF-8 出力を統一

#### Step 6: データ突合
- 入力: エクスポート CSV、登録用 CSV、`contacts.csv`、`contactgroups.csv`
- 出力: `contacts_labels_*.csv`, `skip_list_*.csv`
- 特徴:
  - 氏名と複数メールアドレスを使って照合
  - `people/` の連絡先のみを対象
  - ラベル数ごとに CSV を分割出力

#### Step 7: ラベル登録
- 入力: `contacts_labels_*.csv`
- 出力: Google Contacts 上のラベル付与
- 特徴:
  - ヘッダーからラベル列を自動検出
  - ファイルごとに GAM を実行

### 3.3 ステップ依存関係

現在の GUI では以下を検証する。

- Step 3 を単独で実行する場合:
  既存の連絡先 CSV が必要
- Step 4 を単独で実行する場合:
  既存のラベル CSV が必要
- Step 6 を実行する場合:
  Step 5 で `contacts.csv` と `contactgroups.csv` を取得する必要がある
- Step 7 を実行する場合:
  Step 6 の実行が必要
- Step 3〜7 のいずれかを実行する場合:
  `gam` コマンドが利用可能である必要がある

## 6. 非機能要件

### 6.1 性能
- GUI はワーカースレッドで処理し、応答不能にならないこと
- Python 実行は 5 分、PowerShell 実行は 30 分を上限の目安とする

### 6.2 保守性
- 主要処理を Python / PowerShell のモジュールに分割する
- 実行環境差異を抑えるため、Python 呼び出しは `sys.executable` を使用する
- GAM 実行は `invoke_gam.ps1` に集約する

### 6.3 セキュリティ
- すべてローカル実行を前提とする
- 実行時に扱う CSV やログには個人情報が含まれる可能性がある
- 実行生成物は Git 管理対象に含めない

### 6.4 制約
- Windows 専用
- `gam` 未導入時は Step 3〜7 を実行できない
- Google Contacts の実データを使う運用テストは別途必要
