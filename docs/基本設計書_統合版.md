# Google Contacts 連絡先移行ツール - 統合版 基本設計書

> 注記: この文書の一部記述は初期設計時点のものです。`invoke_gam.ps1` を経由した UTF-8 実行や `contacts.csv` 命名など、現行挙動はソースコードと README を優先してください。

## 1. システム概要

### 1.1 システム名
Google Contacts 連絡先移行ツール - 統合版

### 1.2 目的
旧Googleアカウントから新アカウントへ連絡先を移行する際の全工程（手順2～7）を、1つのGUIアプリケーションで一気通貫で実行できるようにする。

### 1.3 対象ユーザー
企業の社内SE/情シス担当者

### 1.4 開発言語・環境
- **言語**: Python 3.6以降
- **GUI**: tkinter（標準ライブラリ）
- **外部コマンド**: GAM（Google Apps Manager）
- **シェル**: Gitbash（文字化け対策）、PowerShell
- **実行環境**: Windows (MINGW64)

### 1.5 背景・課題
現状、連絡先移行には以下の問題がある：
- 各工程（手順2～7）を個別に手動実行する必要がある
- 実行環境が混在（Python、PowerShell、Gitbash）
- 各工程間のファイル受け渡しが手作業
- GAMコマンドの文字化け問題（特に日本語ラベル名）

### 1.6 解決アプローチ
- **統合GUIアプリケーション**で全工程を自動化
- **Gitbash経由でGAMコマンド実行**により文字化けを回避
- **任意のファイル選択**と**実行前の確認ダイアログ**でユーザビリティ向上
- **実行ログのリアルタイム表示**で進捗を可視化

## 2. 機能要件

### 2.1 主要機能

#### 2.1.1 ファイル選択機能
- **入力ファイル選択**
  - エクスポートCSV（旧アカウントからエクスポートしたCSV）
  - ターゲットメールアドレス（入力欄）
- **出力先選択**
  - 連絡先CSV出力先（任意、デフォルトは自動生成）
  - ラベルCSV出力先（任意、デフォルトは自動生成）
  - ログ出力先（任意、デフォルトは`./logs/`）

#### 2.1.2 処理ステップ選択機能
各工程を個別にON/OFFできるチェックボックス：
- ☑ 手順2: CSV抽出
- ☑ 手順3: 連絡先登録
- ☑ 手順4: ラベル作成
- ☑ 手順5: 登録データ取得
- ☑ 手順6: データ突合
- ☑ 手順7: ラベル登録

#### 2.1.3 実行確認機能
「設定を確認」ボタン押下時に、以下の内容を表示：
- 入力ファイルパス
- ターゲットメールアドレス
- 出力先設定
- 実行する処理ステップ
- 最終確認ダイアログ（はい/いいえ）

#### 2.1.4 自動実行機能
「処理開始」ボタン押下時に、選択された工程を順次自動実行：
- Step 2: CSV抽出（Python）
- Step 3: 連絡先登録（PowerShell → GAM）
- Step 4: ラベル作成（Gitbash → GAM）
- Step 5: 登録データ取得（Gitbash → GAM）
- Step 6: データ突合（Python）
- Step 7: ラベル登録（Gitbash → GAM）

#### 2.1.5 ログ表示機能
- 実行ログをGUI上にリアルタイム表示
- タイムスタンプ付きログメッセージ
- 進捗状況の可視化（✅成功、⏳処理中、❌失敗）

#### 2.1.6 エラーハンドリング機能
- 各ステップでエラー発生時に処理を中断
- エラー内容をログとダイアログに表示
- 中間ファイルの保存（デバッグ用）

#### 2.1.7 Gitbash検出機能
- システム起動時にGitbashの有無を自動検出
- Gitbash未検出時に警告メッセージ表示
- フォールバック機能（PowerShell UTF-8版）

## 3. 非機能要件

### 3.1 パフォーマンス
- **処理速度**: 100件の連絡先を10分以内に処理
- **応答性**: GUIの応答速度は1秒以内

### 3.2 ユーザビリティ
- **直感的なUI**: 日本語ラベル、明確なグループ化
- **確認ダイアログ**: 誤操作防止のための念押し確認
- **ログの可読性**: タイムスタンプ、絵文字アイコン、色分け

### 3.3 互換性
- **Python**: 3.6以降
- **OS**: Windows 10以降
- **GAM**: 最新版（UTF-8出力対応）
- **Gitbash**: Git for Windows 2.x以降

### 3.4 セキュリティ
- すべての処理はローカルで完結
- ファイルデータはメモリ上でのみ処理
- ネットワーク通信はGAMコマンドのみ
- 中間ファイルは実行ディレクトリ内に保存

### 3.5 保守性
- **モジュール化**: 既存のPythonモジュールを再利用
- **ログ保存**: 実行ログをファイルに保存（デバッグ用）
- **バージョン管理**: Git管理、変更履歴の記録

### 3.6 拡張性
- **処理ステップの追加**: 新しいステップを容易に追加可能
- **設定ファイル**: 将来的に設定の外部化が可能

## 6. エラー処理

### 6.1 入力ファイルエラー
- **エラー**: ファイルが存在しない、読み込み権限がない
- **処理**: エラーダイアログ表示、処理中断
- **ログ**: `❌ エラー: 入力ファイルが見つかりません: {ファイルパス}`

### 6.2 Gitbash未検出エラー
- **エラー**: Gitbash (bash.exe) が見つからない
- **処理**: 警告ダイアログ表示、PowerShell UTF-8版にフォールバック
- **ログ**: `⚠️ Gitbash未検出、PowerShellで実行します`

### 6.3 GAMコマンドエラー
- **エラー**: GAMコマンド実行失敗（認証エラー、API制限など）
- **処理**: エラーメッセージをログに記録、処理中断
- **ログ**: `❌ Step X: 失敗 - {エラー内容}`

### 6.4 Python実行エラー
- **エラー**: Pythonスクリプト実行失敗（モジュールなし、構文エラーなど）
- **処理**: エラーメッセージをログに記録、処理中断
- **ログ**: `❌ Step X: 失敗 - {エラー内容}`

### 6.5 PowerShell実行エラー
- **エラー**: PowerShellスクリプト実行失敗
- **処理**: エラーメッセージをログに記録、処理中断
- **ログ**: `❌ Step 3: 失敗 - {エラー内容}`

### 6.6 中間ファイルエラー
- **エラー**: 中間ファイルの生成失敗、読み込み失敗
- **処理**: エラーメッセージをログに記録、処理中断
- **ログ**: `❌ 中間ファイルエラー: {ファイルパス}`

## 8. 制約事項

### 8.1 技術的制約
- Python 3.6以降が必要
- Gitbash推奨（文字化け対策）
- GAMがインストール済みであること
- Windows環境でのみ動作確認

### 8.2 機能的制約
- 手順1（旧アカウントからのエクスポート）は手動実行
- Google Workspace（旧G Suite）アカウント必須
- GAMの認証設定が事前に完了していること

### 8.3 運用的制約
- 大量データ（1000件以上）の処理は未検証
- API制限によりエラーが発生する可能性あり
- 実行中はGUI操作不可（別スレッド実行）
